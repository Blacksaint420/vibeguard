# Support

Use the public issue tracker for:

- Bug reports with reproduction steps.
- Feature requests and rule ideas.
- Documentation corrections.
- CI or GitHub Action integration questions.

Before opening an issue, include the VibeGuard version, Node.js version, operating system, command used, and the smallest safe reproduction you can share.

Do not include secrets, private source code, proprietary prompts, AI BOM output, or full scan reports in public issues. Report suspected vulnerabilities privately through `SECURITY.md`.
