# Code of Conduct

VibeGuard aims to be a practical, respectful open-source project for developers and security teams.

## Expected Behavior

- Be direct, factual, and constructive.
- Assume contributors are acting in good faith.
- Keep discussion focused on the technical issue, user impact, and project goals.
- Give security reports and sensitive findings extra care and discretion.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing another person's private information.
- Publicly disclosing suspected vulnerabilities before maintainers have had a reasonable opportunity to respond.
- Repeated off-topic comments that disrupt issue or pull request work.

## Enforcement

Maintainers may edit or remove comments, close issues, block accounts, or take other reasonable steps to protect the project and its contributors. Security-related conduct issues can be reported through the private vulnerability reporting process in `SECURITY.md`.
